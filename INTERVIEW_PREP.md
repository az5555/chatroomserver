# ChatServer 面试准备指南

你好！我是面试官。作为 ChatServer 项目的开发者，为了在面试中展现你的深度和技术理解，请务必吃透以下内容。本项目是一个基于 C++ 实现的高性能 Web 服务器/聊天服务器。

---

## 1. 项目亮点 (Highlights)
*这些是你应该主动抛出来的“卖点”：*

*   **高并发 I/O 模型**：基于 `epoll` 的边缘触发 (ET) + 非阻塞 I/O，极大地提升了处理连接的能力，能够轻松应对大量同时在线的客户端。
*   **线程池机制**：通过封装 `ThreadPool`，实现了任务队列与工作线程的解耦，有效降低了高并发下的线程创建与销毁开销。
*   **资源管理与 RAII**：充分利用 C++ RAII 技术（如 `sqlconnRAII`），确保了数据库连接、内存和文件描述符等资源的自动释放，有效防止内存泄漏和僵尸连接。
*   **数据库连接池**：为了避免频繁建立/断开 MySQL 连接，实现了连接池技术，显著优化了高负载下的数据库请求性能。
*   **高性能定时器**：基于大顶堆实现的 `HeapTimer`，用于精准处理连接超时，复杂度为 $O(1)$ 或 $O(\log n)$，比轮询高效得多。

---

## 2. 技术难点与解决思路 (Difficulties)
*面试官喜欢听这里，准备好具体的调试经历：*

*   **Epoll 边缘触发（ET）模式下的粘包/拆包与数据读取**：
    *   **难点**：ET 模式下必须一次性读取缓冲区所有数据，否则会导致数据丢失或死锁。
    *   **解决方法**：设计了自定义的 `Buffer` 类，支持动态扩容，通过循环读取直到 `read` 返回 `EAGAIN`，同时实现了完善的应用层协议处理（`ProtocolRequest` / `ProtocolResponse`）。
*   **多线程同步与竞态条件**：
    *   **难点**：数据库连接池和线程池在多线程环境下的临界区保护。
    *   **解决方法**：合理使用锁（`std::mutex`, `std::unique_lock`）和条件变量（`std::condition_variable`），严格控制临界区粒度。
*   **内存管理与 buffer 复用**：
    *   **难点**：频繁分配内存会导致内存碎片，影响服务器长期运行的性能。
    *   **解决方法**：封装了 `Buffer` 类，尽可能复用内存，并结合智能指针管理对象生命周期。

## 3. 高频面试考点 (Exam Points)
*这些是面试官必问的，请务必掌握：*

### 网络编程基础
*   **Epoll 的原理与实现**：为什么比 Select/Poll 高效？（关键点：红黑树、就绪链表、回调机制）。
*   **ET (边缘触发) 与 LT (水平触发) 的区别**：处理方式有何不同？
*   **TCP 三次握手/四次挥手**：特别关注状态转移，例如 `TIME_WAIT` 的作用。

### 并发与多线程
*   **线程池的工作原理**：如何处理任务队列？如何处理线程间的负载均衡？
*   **锁的机制**：互斥锁、自旋锁、读写锁的区别与适用场景。
*   **死锁的预防**：如何检测和避免死锁？

### C++ 技术栈
*   **RAII (Resource Acquisition Is Initialization)**：它的核心思想是什么？如何应用在数据库连接池中？
*   **智能指针**：`shared_ptr`, `unique_ptr` 在项目中的具体应用。
*   **移动语义 (std::move)**：如何减少对象的拷贝开销？

### 系统架构
*   **数据库连接池如何工作**：如何实现连接的获取与归还？如何处理连接失效？
*   **协议设计**：如果需要兼容 HTTP 或其他协议，应该如何重构 `ProtocolRequest` 和 `ProtocolResponse`？

## 4. 给面试官的“绝杀”回答建议
*不要只回答定义，要结合项目：*

*   **问：为什么要用连接池？**
    *   **答**：在本项目高并发场景下，直接操作 MySQL 会因为频繁的连接建立造成巨大延迟。连接池提前创建连接，减少了连接建立带来的耗时，直接提升了服务器整体的 QPS。
*   **问：定时器为什么要用堆？不用链表？**
    *   **答**：本项目连接数可能很大。使用双向链表查找最小超时连接复杂度为 $O(n)$，在大规模连接下会卡住主线程。使用大顶堆，将超时时间排序，处理超时只需 $O(1)$，调整堆结构只需 $O(\log n)$，性能差异巨大。
